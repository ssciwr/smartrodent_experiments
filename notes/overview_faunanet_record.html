faunanet-record Codebase Overview

faunanet-record is a Python package and CLI for chunked microphone recording. It creates configuration/data directories, merges YAML configuration, records audio through PyAudio/PortAudio, writes WAV files, and exposes Recorder and Runner for library use.

Dominant coding and testing paradigms

Coding style

Small object-oriented core around RecorderBase, Recorder, and Runner, with procedural helper functions for setup, YAML loading, and dictionary merging. Runtime behavior is configuration-driven through nested dictionaries loaded from YAML.

Testing style

Pytest-based unit and integration tests. Tests exercise real PyAudio behavior against virtual audio devices in CI rather than using heavy mocking. Autouse fixtures create and remove config/data directories under platform-specific user config/data locations.

Operational model

The CLI first installs default configs and data folders, then each run creates a timestamped output subdirectory, optionally dumps the effective YAML config, and records sequential WAV chunks until a stop condition is met.

CLI usage

faunanet_record install --cfg_dir=src/faunanet_record
faunanet_record get-device-info
faunanet_record run
faunanet_record run --cfg=path/to/record_config.yml
faunanet_record run --replace='{"Recording":{"length_s":2}}'

Click converts the Python function get_device_info to the CLI command get-device-info. The package script name in pyproject.toml is faunanet_record.

Library usage

from faunanet_record import Runner, Recorder

runner = Runner({"Output": {"runtime": 8}})
runner.run()

Run tests locally

Audio tests require PortAudio and an available input device. To mirror CI on Ubuntu, start virtual PulseAudio devices before pytest:

sudo apt-get install -y portaudio19-dev pulseaudio
pulseaudio -D
pactl load-module module-null-sink sink_name=virtspk sink_properties=device.description=Virtual_Speaker
pactl load-module module-null-sink sink_name=virtmic sink_properties=device.description=Virtual_Microphone_Sink
pactl load-module module-remap-source master=virtmic.monitor source_name=virtmic source_properties=device.description=Virtual_Microphone
python3 -m pytest --cov=./src/faunanet_record --cov-report=xml

pyproject.toml sets addopts = -v -x, testpaths = ["tests"], branch coverage on src/faunanet_record, and a coverage threshold of 80%.

Constraints, risks, and open questions

  • Audio hardware dependency: normal operation requires PortAudio/PyAudio and a working input device. CI works around this with virtual PulseAudio sources.
  • Linux-centric support: package classifiers and CI target POSIX/Linux. README mentions macOS success but no Windows validation.
  • Hardcoded sample format: recording uses pyaudio.paInt16. README explicitly notes this as a current limitation.
  • Chunk divisibility: (sample_rate * length_s) / chunk_size is truncated to an integer in stream_audio; README warns that non-divisible settings can cause PortAudio frame issues.
  • Configuration drift: README documents runtime: "inf", but Runner.run only handles None, int, and datetime. A string runtime may not start recording.
  • Folder collisions: Runner creates timestamped folders with second-level precision using mkdir() without exist_ok=True; repeated starts in the same second can collide.
  • Resource cleanup: cleanup relies partly on __del__ and internal _close(); explicit lifecycle management may be safer for long-running deployments.
  • Public API naming: README contains both hyphenated and underscored command spellings. pyproject.toml defines faunanet_record; Click subcommands use hyphens for underscores.
