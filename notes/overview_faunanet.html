faunanet Codebase Overview

faunanet is a Python bioacoustics package and REPL-driven watcher service. It monitors an input directory, loads configurable audio preprocessors and neural-network models, analyzes new audio files, optionally restricts predictions by species presence, and writes CSV results plus runtime configuration snapshots.

Dominant coding and testing paradigms

  • Object-oriented runtime orchestration: Watcher, Recording, ModelBase, PreprocessorBase, and SpeciesPredictorBase are the central extension and orchestration objects.
  • Configuration-driven composition: YAML files decide directories, model names, analyzer parameters, recording options, and species predictor options.
  • Plugin-style model loading: downloaded or custom model folders provide model.py, preprocessor.py, model weights, labels, and a model-specific default.yml.
  • Event/process-driven application behavior: watchdog.Observer runs in a separate process; REPL commands start, stop, pause, resume, restart, and reconfigure the watcher.
  • Pytest-based integration-heavy tests: fixtures create temporary installations, download/cache model assets, run watcher and REPL flows, and compare generated CSVs with stored expected results.

Class and function descriptions

ModelBase in src/faunanet/model_base.py

  • Abstract base class for model plugins.
  • Validates model and label paths, loads labels, dispatches model loading by filename, and converts chunk predictions into time-windowed result dictionaries.
  • Subclasses implement predict(data).

PreprocessorBase in src/faunanet/preprocessor_base.py

  • Abstract base class for audio preprocessing plugins.
  • Reads audio with the configured sample rate and stores chunk metadata for the model.
  • Subclasses implement process_audio_data(rawdata).

Recording in src/faunanet/recording.py

  • Extends birdnetlib.main.RecordingBase.
  • Binds one preprocessor, one model, one file path, confidence thresholds, and optional species-presence filtering.
  • Checks component compatibility through shared name values, reads/preprocesses audio, and exposes BirdNET-style detections.

SpeciesPredictorBase in src/faunanet/species_predictor.py

  • Extends birdnetlib.species.SpeciesList.
  • Loads species_presence_model.tflite and labels.txt from a model directory.
  • Predicts present species for latitude, longitude, and date; supports file cache export and cache clearing.

Watcher and AnalysisEventHandler in src/faunanet/watcher.py

  • Watcher owns runtime directories, model config, recording config, species predictor config, multiprocessing state, and output cleanup.
  • AnalysisEventHandler receives filesystem creation events and calls Watcher.analyze for files matching the configured suffix.
  • watchertask(watcher) creates the watchdog observer inside the worker process.

FaunanetCmd in src/faunanet/repl.py

  • Interactive command loop registered as the faunanet console script.
  • Processes --name=value command arguments, merges default and custom YAML, controls watcher lifecycle, and reports errors/status.

Setup functions in src/faunanet/faunanet_setup.py

  • make_directories creates home, model, output, local config, local cache, and sample-data directories.
  • download_model_files fetches default model packages from Hugging Face through pooch.
  • download_example_data fetches sample audio assets.
  • set_up orchestrates installation setup from YAML.

Utility functions in src/faunanet/utils.py

  • YAML reading and nested dictionary updates.
  • TensorFlow Lite and TensorFlow SavedModel loading.
  • Dynamic import helpers for model/preprocessor plugin modules.
  • File-completion polling for incoming audio files.

Starting reading points and runtime entry points

Best first files

  1. README.md for purpose, setup, usage, Docker, and hosted docs.
  2. src/faunanet/watcher.py for the central runtime path.
  3. src/faunanet/repl.py for command behavior and config merging.
  4. src/faunanet/recording.py, model_base.py, and preprocessor_base.py for extension contracts.
  5. tests/watcher_test.py and tests/repl_test.py for end-to-end expected behavior.

Main code paths

  • Console start: faunanet calls faunanet.repl:run, which starts FaunanetCmd.cmdloop().
  • Setup: set_up --cfg=... calls faunanet_setup.set_up to create folders and download default assets.
  • Watch and analyze: start --cfg=... merges config, creates Watcher, starts a subprocess, schedules AnalysisEventHandler, and writes results_*.csv.
  • Model exchange: change_analyzer updates model/preprocessor classes from another model folder while preserving watcher state.

CI and release entry points

  • .github/workflows/main.yml runs tests on Ubuntu, macOS, and Windows for Python 3.9 and 3.12, installs ffmpeg, installs .[tensorflow], installs development requirements, and uploads coverage from the Ubuntu Python 3.12 job.
  • .github/workflows/cd.yml builds distributions, installs the generated source distribution, runs an import smoke test, publishes to PyPI on tags, signs artifacts, and creates a GitHub release.

How to run, test, and build docs

Install for development

  • Install ffmpeg first: Debian/Ubuntu sudo apt install ffmpeg, macOS brew install ffmpeg, Windows choco install ffmpeg.
  • Create and activate a virtual environment, then run python3 -m pip install -e .[tensorflow,dev].
  • Use .[tensorflow-lite,dev] only when TensorFlow Lite support is sufficient for the target model.

Run the application

  • Start the REPL with faunanet.
  • Initial setup from the REPL: set_up --cfg=path/to/install_config.yml.
  • Start watching from the REPL: start --cfg=path/to/analysis_config.yml.
  • Useful REPL commands include help, status, pause, continue, restart, change_analyzer, cleanup, stop, and exit.

Run tests

  • Standard test command: pytest.
  • CI-equivalent coverage command: pytest --cov=./src/faunanet --cov-report=xml.
  • Targeted examples: pytest tests/watcher_test.py, pytest tests/repl_test.py, pytest tests/recording_test.py.
  • Pytest defaults in pyproject.toml are -v -x and testpaths = ["tests"].

Build documentation

  • Install documentation extras: python3 -m pip install -e .[doc].
  • Build from the documentation directory: cd doc && make html.
  • Windows build helper: doc/make.bat.

Constraints, risks, and open questions

  • Dynamic plugin imports: utils.load_module and load_name_from_module execute Python code from model directories. This enables flexible custom models but requires trusted model folders and strong runtime error handling.
  • External downloads: setup and tests rely on model/audio assets fetched from Hugging Face through pooch. Network availability and cache state can affect first-run behavior.
  • Runtime TensorFlow choice: either tensorflow or tflite-runtime must be installed for actual inference. The correct extra depends on model format.
  • Cross-platform shell details: CI runs all supported platforms, but README activation snippets differ between Unix shells and Windows shells.
  • Potential release workflow mismatch: .github/workflows/cd.yml smoke tests import faunanet_record, while this package exports faunanet.
  • Potential method-name mismatch: scout findings indicate Watcher.change_analyzer() references restore_old_state while the implemented helper is named _restore_old_state.
  • Potential config naming mismatch: default and runtime paths should be checked around SpeciesPresence versus SpeciesPredictor naming before changing species-filter configuration.
  • Filesystem watcher timing: incoming audio handling depends on file-completion polling and process state. Large files, slow writes, or abrupt process termination can affect cleanup and result consistency.
  • Output cleanup risk: watcher cleanup intentionally manipulates timestamped output directories; changes should preserve data-loss safeguards and tests around cleanup limits.
  • API inheritance boundary: Recording and SpeciesPredictorBase extend birdnetlib classes. Upstream API changes can affect constructor expectations, detection formats, and species list semantics.
